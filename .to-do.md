## 🔴 Bugs & Broken Things

- [x] **Typo in electron-store schema** — `main.ts:27` has `langage: 'en'` → should be `language: 'en'`. This means the default stored setting key doesn't match the `UserSettings.language` type.
- [x] **`window.electron` is loosely typed** — `global-types.d.ts:213` declares `electron: { [key: string]: any }`. This means zero type-safety for any IPC call. Should be a typed interface matching `preLoad.cts`.
- [ ] **Print items table is hardcoded** — `printService.ts:59–64` still has Cat/Dog/Horse/Pig placeholder data instead of mapping `bill.purchases` or querying actual items for the bill.
- [ ] **`onTabChange` listener leaks** — `mainPage.tsx:17` registers an `ipcRenderer.on(...)` listener in `useEffect` but never returns a cleanup function. Each re-render/re-mount adds a duplicate listener.
- [x] **Double `LanguageProvider`** — `main.tsx:12` wraps `<App />` with `<LanguageProvider>`, and then `mainPage.tsx:45` wraps `<MainPage />` again. The inner one shadows the outer, wasting a context layer.
- [x] **`CollectionRepository` saves booleans as TEXT** — `collection.tsx:27-28` uses JS `false`, but the DB schema (`connection.ts:146-147`) defines `delivery_status TEXT` and `collection_status TEXT`. The mismatch means booleans arrive in SQLite as `0`/`1` but are compared as strings elsewhere.
- [x] **`App.tsx` has dead code** — Lines 2, 6-8 contain commented-out `DirectionProvider` imports. Clean up or re-enable.
- [ ] **`records_columns.tsx:20` date parsing can crash** — `format(new Date(info.getValue() as string), ...)` will throw on malformed dates. Needs a try/catch or safe parser.
